Food Pop-up Business Model

Skills: SQL, Logical Design, ER Diagram

Objectives: Design a database from scratch for a business model called Food Popup that connects Cal Poly students to local food popups in San Luis Obispo area.

Contents: The workflow of this project can be described as the following:

  1. Develop comprehensive business rules that inform the model
  2. Create definitions that define the entities for the entire database
  3. Design Enhanced Entity-Relationship (EER) diagram
  4. Create a logical design that explains a series logical relationships called entities and attributes
  5. Use SQL to create database, populate database
  6. Execute many SQL queries to answer potential business questions
